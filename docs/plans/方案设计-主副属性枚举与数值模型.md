# 主/副属性枚举与数值模型 — 细化方案

> 基于《方案设计-职业属性与成长系统.md》，进一步细化属性枚举定义、数值来源、growth 配置加载和运行时计算逻辑。
> 日期：2026-05-31

---

## 一、枚举定义

### 1.1 PrimaryAttribute（主属性，4 个）

```kotlin
// 新增: api/attribute/PrimaryAttribute.kt

enum class PrimaryAttribute(
    val displayName: String,  // 中文名，供 UI/HUD 展示
    val shortName: String     // 短标识，透传给 AttributeDriver
) {
    STRENGTH    ("力量", "STR"),
    INTELLIGENCE("智力", "INT"),
    AGILITY     ("敏捷", "AGI"),
    ENDURANCE   ("耐力", "END");
}
```

- 4 个主属性可被玩家自由分配点数
- 每个职业有 1 个 `main-attribute`，决定伤害公式走哪条线
- `shortName` 直接对接 `AttributeDriver` 的属性 ID，保证与 AttributePlus 等外部插件兼容

### 1.2 SubAttribute（子属性，6 个）

```kotlin
// 新增: api/attribute/SubAttribute.kt

enum class SubAttribute(
    val displayName: String,
    val shortName: String
) {
    CRIT      ("暴击", "CRIT"),
    STRIKE    ("强击", "STRIKE"),
    FAITH     ("信仰", "FAITH"),
    HASTE     ("急速", "HASTE"),
    TOUGHNESS ("坚韧", "TOUGH"),
    SPIRIT    ("精神", "SPIR");
}
```

- **子属性不可自由分配**，仅由 growth + 装备 + Buff 提供
- 文档约束：**无"闪避"子属性**，枚举中不包含

### 1.3 为何不定义 JobType / ClassType 枚举

职业身份由 `ImmutableJob.id`（YAML 文件名）决定，不是硬编码枚举。好处：

- 新增职业/转职不需改代码枚举，加 YAML 即可
- `main-attribute` 字段挂在 Job YAML 上，灵活指定
- 与现有 `Registries.JOB` 加载机制一致

## 二、数值来源模型

### 2.1 最终属性值叠加公式

```
最终属性 = Lv1基础值           ← Job YAML  attributes.primary/sub
         + growth × (Lv-1)     ← growth YAML  per-level
         + 每N级额外           ← growth YAML  every-n-levels
         + 自由分配点          ← Player metadata  attribute.invested.XXX
         + 装备加成            ← 后续系统（装备宝珠/武器等）
         + Buff/Debuff         ← 后续系统
```

**运行时实时计算，不缓存最终值到数据库。**

### 2.2 各职业 Lv1 基础属性数值

### 2.5 自由属性点规则

```
每级 +3 自由点
可分配目标: STRENGTH, INTELLIGENCE, AGILITY, ENDURANCE（仅 4 个主属性）
洗点消耗: 5000 金币（在 class/settings.yml 配置）
```

- 自由点是**跨阶段累积**：基础流派和分支流派共用同一套
- 洗点只返还自由属性点，不触及技能树

### 2.6 HP/MP/SP/ULP 计算公式

```yaml
# class/settings.yml → formula 节点，JS 表达式
max-hp:  "class.endurance * 12 + class.hpPerLevel * player.level + class.baseHP"
max-mp:  "class.spirit * 8 + class.mpPerLevel * player.level + class.baseMP"
max-sp:  "class.endurance * 6 + class.spPerLevel * player.level + class.baseSP"
max-ulp: "500"   # 固定值，全局统一
```

- JS 变量 `class.xxx` 由 `ClassAttributeManager` 注入
- `player.level` 来自 `PlayerRouter`
- 与现有 `magic-point.upper-limit.expression` 同样的 JS 表达式模式

其他 7 个 Job YAML（guardian/assassin/archer/mage/swordsman/blade-master/grand-master）按上表数值同步修改。

### 3.2 growth 配置文件（新增 6 个）

```yaml
# resources/class/growth/warrior.yml
id: warrior_growth
per-level:
  HP: 28
  MP: 8
  SP: 5
  primary:
    STRENGTH: 3
    ENDURANCE: 2
    AGILITY: 1
  sub:
    STRIKE: 1
```

`damage-factor` 为主属性对最终伤害的转换系数，预留字段，等统一伤害管线时启用。

## 四、PlayerTemplate Metadata 约定

```
attribute.main-attribute     → "STRENGTH"                 # String，首次选职业时写入
attribute.free-points        → 7                          # Int，当前剩余自由属性点
attribute.invested.STRENGTH  → 15                         # Int，在力量上已分配点数
attribute.invested.INTELLIGENCE → 2                       # Int
attribute.invested.AGILITY   → 5                          # Int
attribute.invested.ENDURANCE → 3                          # Int
```

- 仅主属性可 invest，子属性不可分配
- 洗点时：清零所有 `invested.*`，计算返还总额写入 `free-points`
- 这些 key 不硬编码为常量，在 `ClassAttributeManager` 中通过 `metadataKey("invested", attr)` 拼装

## 六、核心类接口设计

### 6.1 ClassGrowthConfig

```kotlin
// 解析 class/growth/*.yml 的单个 section，如 "warrior_growth:"
class ClassGrowthConfig(section: ConfigurationSection) : Unique {

    override val id: String                              // = section.name，如 "warrior_growth"

    val hpPerLevel: Int                                  // 每级 HP 成长
    val mpPerLevel: Int                                  // 每级 MP 成长
    val spPerLevel: Int                                  // 每级 SP 成长

    val primaryPerLevel: Map<PrimaryAttribute, Int>      // 每级主属性成长
    val subPerLevel: Map<SubAttribute, Int>              // 每级子属性成长
    val subEveryNLevels: Map<Int, Map<SubAttribute, Int>>  // 每 N 级额外子属性

    val damageFactor: Double                             // 主属性→伤害系数

    companion object {
        // 从 __option__.growth 引用字符串查找，如 "warrior_growth"
        fun getOrNull(growthId: String): ClassGrowthConfig? {
            return Registries.CLASS_GROWTH.getOrNull(growthId)
        }
    }
}
```

**加载方式**：`createDeepMultiBuiltin("class/growth", ...)` → `Registries.CLASS_GROWTH`，与 `SKILL_TREE` 同模式。

**无 growth 的情况**：如果 Job YAML 未写 `growth:` 字段，`ClassAttributeManager` 对 growth 部分返回 0，不影响运行时。

### 6.2 ImmutableJob 扩展

在现有 `ImmutableJob` 中新增 5 个惰性解析属性：

```kotlin
// 以下为现有 ImmutableJob 的新增属性
// 全部通过 option.getXxx() 惰性解析，首次访问后缓存在字段

/** 主属性类型，null 表示未配置（非职业 Job） */
val mainAttribute: PrimaryAttribute?
    get() {
        val raw = option.getString("main-attribute") ?: return null
        return try {
            PrimaryAttribute.valueOf(raw.uppercase())
        } catch (_: Exception) {
            null
        }
    }

/** Lv1 基础主属性 */
val basePrimary: Map<PrimaryAttribute, Int>
    get() {
        val section = option.getConfigurationSection("attributes.primary") ?: return emptyMap()
        return PrimaryAttribute.entries.associate { attr ->
            attr to section.getInt(attr.name, 0)
        }
    }

/** Lv1 基础子属性 */
val baseSub: Map<SubAttribute, Int>
    get() {
        val section = option.getConfigurationSection("attributes.sub") ?: return emptyMap()
        return SubAttribute.entries.associate { attr ->
            attr to section.getInt(attr.name, 0)
        }
    }

/** Lv1 基础资源 */
val baseResource: Map<String, Int>
    get() {
        val section = option.getConfigurationSection("attributes.resource") ?: return emptyMap()
        return mapOf(
            "HP" to section.getInt("HP", 0),
            "MP" to section.getInt("MP", 0),
            "SP" to section.getInt("SP", 0)
        )
    }

/** growth 配置 ID */
val growthId: String?
    get() = option.getString("growth")
```

### 6.3 ClassAttributeManager

```kotlin
object ClassAttributeManager {

    // ==================== 初始化 ====================

    /** 首次选职业时调用，写入 metadata 初始值 */
    fun init(template: PlayerTemplate, job: ImmutableJob)

    // ==================== 升级处理 ====================

    /** 监听 PlayerLevelChangeEvent，发放自由属性点 */
    fun onLevelUp(template: PlayerTemplate, newLevel: Int)

    // ==================== 自由属性点 ====================

    /** 分配自由点到指定主属性，返回是否成功 */
    fun allocate(template: PlayerTemplate, attr: PrimaryAttribute, points: Int): Boolean

    /** 获取当前可用自由点数 */
    fun getFreePoints(template: PlayerTemplate): Int

    /** 洗点：清零 invested.*，返还全部到 free-points，扣除金币 */
    fun respec(template: PlayerTemplate): Boolean

    // ==================== 实时查询（运行时计算，不缓存） ====================

    /** 查询最终主属性值 */
    fun getPrimary(template: PlayerTemplate, attr: PrimaryAttribute): Int

    /** 查询最终子属性值 */
    fun getSub(template: PlayerTemplate, attr: SubAttribute): Int

    /** 查询 HP/MP/SP/ULP 上限 */
    fun getMaxHP(template: PlayerTemplate): Int
    fun getMaxMP(template: PlayerTemplate): Int
    fun getMaxSP(template: PlayerTemplate): Int
    fun getMaxULP(): Int { return 500 }  // 固定值

    /** 查询主属性类型 */
    fun getMainAttribute(template: PlayerTemplate): PrimaryAttribute?

    // ==================== 内部 ====================

    /** 获取当前职业 Job */
    private fun getJob(template: PlayerTemplate): ImmutableJob?

    /** 获取当前 growth 配置 */
    private fun getGrowth(template: PlayerTemplate): ClassGrowthConfig?

    /** 计算指定主属性的最终值 */
    private fun calcPrimary(template: PlayerTemplate, attr: PrimaryAttribute): Int {
        val job = getJob(template) ?: return 0
        val growth = getGrowth(template)
        val level = template.getLevel()
        val invested = getInvested(template, attr)

        val base = job.basePrimary[attr] ?: 0                           // Lv1 基础
        val perLevel = growth?.primaryPerLevel?.get(attr) ?: 0          // 每级
        val everyN = calcEveryN(template, growth, attr)                 // 每 N 级额外
        return base + perLevel * maxOf(0, level - 1) + everyN + invested
    }

    /** 计算指定子属性的最终值 */
    private fun calcSub(template: PlayerTemplate, attr: SubAttribute): Int {
        val job = getJob(template) ?: return 0
        val growth = getGrowth(template)
        val level = template.getLevel()

        val base = job.baseSub[attr] ?: 0
        val perLevel = growth?.subPerLevel?.get(attr) ?: 0
        val everyN = calcEveryN(template, growth, attr)
        return base + perLevel * maxOf(0, level - 1) + everyN
    }

    /** 计算每 N 级额外 */
    private fun calcEveryN(template: PlayerTemplate, growth: ClassGrowthConfig?, attr: Any): Int

    /** 从 metadata 读取 invested 值 */
    private fun getInvested(template: PlayerTemplate, attr: PrimaryAttribute): Int

    /** 获取自由属性点总额（calcAccumulated） */
    private fun calcTotalFreePoints(level: Int): Int
}
```

### 6.5 ClassAttributeListener

```kotlin
object ClassAttributeListener {

    @SubscribeEvent
    fun onRouteSet(e: PlayerSetRouteEvent.Post) {
        val template = e.template
        val job = e.route.getJob()
        // 首次选职业（metadata 中无 main-attribute）且有 growth → 初始化
        if (job.growthId != null && template["attribute.main-attribute"] == null) {
            ClassAttributeManager.init(template, job)
        }
        AttributeProvider.refresh(e.player)
    }

    @SubscribeEvent
    fun onLevelChange(e: PlayerLevelChangeEvent) {
        // 升级发放自由属性点并同步属性
        ClassAttributeManager.onLevelUp(e.template, e.to)
        AttributeProvider.refresh(e.player)
    }
}
```

## 七、与现有系统对接

### 7.1 AttributeProvider 扩展

在现有 `AttributeProvider.getAttributeList()` 末尾追加：

```kotlin
private fun appendClassAttributes(player: Player, list: MutableList<String>) {
    val template = player.plannersTemplate

    for (attr in PrimaryAttribute.entries) {
        val v = ClassAttributeManager.getPrimary(template, attr)
        if (v > 0) {
            list.add("${attr.shortName}: +$v")
        }
    }
    for (attr in SubAttribute.entries) {
        val v = ClassAttributeManager.getSub(template, attr)
        if (v > 0) {
            list.add("${attr.shortName}: +$v")
        }
    }
    list.add("MAX_HEALTH: +${ClassAttributeManager.getMaxHP(template)}")
}
```

属性字符串通过 `AttributeDriver.set(player, "planners-class", list, -1)` 推送到外部属性插件。

```yaml
settings:
```

`classAttribute` 为 `ClassAttributeAPI` 在 JS 引擎中的注册名。

## 八、与文档约束对照

| 文档要求 | 方案处理 |
|---------|---------|
| 六大职业枚举 | 不硬编码，由 Job YAML `main-attribute` 决定身份 |
| 4 主属性 | ✅ `PrimaryAttribute` 枚举 |
| 6 子属性（无闪避） | ✅ `SubAttribute` 枚举 |
| 职业固定成长表 | ✅ 6 份 `class/growth/*.yml` |
| 自由点每级 +3 | ✅ `class/settings.yml` → `free-points.per-level` |
| HP/MP/SP 公式 | ✅ JS 表达式在 `class/settings.yml` → `formula` |
| ULP = 500 | ✅ `getMaxULP()` 固定返回 500 |
| 护盾系统 | ❌ 不在此模块，归属统一伤害管线(3.5) |
| 武器/后续系统技能点 | ❌ 不在此模块，由对应子系统管理 |
| 禁止百分比属性 | ✅ growth 全固定值 |
