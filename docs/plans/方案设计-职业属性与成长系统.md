# 职业属性与成长系统 — 技术方案（V2）

> 基于 planners-v2 现有架构，Job = 职业，不引入额外的 ClassType

---

## 一、现有基础

```
Registries.JOB  ← 所有 Job 定义（YAML 加载为 ImmutableJob）
  ├─ warrior    (战士)
  ├─ guardian   (守护者/骑士)
  ├─ assassin   (刺客)
  ├─ archer     (射手/弓箭手)
  ├─ mage       (法师)
  ├─ swordsman  (剑士)
  ├─ blade-master (剑魂)
  └─ grand-master (剑圣)

Registries.ROUTER  ← 职业路线（决定 Job 之间的转职关系）
  └─ soldier: swordsman → blade-master → grand-master

PlayerTemplate  ← 玩家数据（MetadataContainer）
  └─ route: PlayerRoute  ← 当前 Job + 已学 Skill
```

### 已有能力

| 能力 | 实现 |
|------|------|
| Job 定义 | `ImmutableJob` — 名称、技能列表、变量 |
| 技能 Hook 属性 | `hook.attributes` — 字符串列表透传给 `AttributeDriver` |
| 等级系统 | `AlgorithmLevel` — JS 表达式经验曲线，ALL/ROUTER/JOB 隔离 |
| 法力值 | `settings.magic-point` — MP 上限/恢复表达式 |
| 属性驱动 | `AttributeDriver` — 设置/移除实体属性值 |
| 属性聚合 | `AttributeProvider` — 从 Job + Skill 汇总属性字符串列表 |

### 缺失

- Job 没有结构化属性定义（Lv1 基础值、每级成长）
- 没有主属性/子属性区分
- 没有自由属性点
- 没有职业技能点
- HP/SP 换算未实现

## 二、设计思路

**核心原则：Job 即职业，属性配置直接挂在 Job YAML 上。**

```
Job YAML（扩展后）              独立配置
──────────────────────         ───────────────────
__option__:                    class/growth/
  name: 战士                       warrior.yml
  main-attribute: STRENGTH         knight.yml
  attributes:                      assassin.yml
    primary: { ... }               ...
    sub: { ... }               class/settings.yml
    resource: { ... }              (全局规则)
  growth: warrior_growth ───→ 引用 growth 配置
  skill: [ ... ]
```

- **Job 自带属性**：Lv1 基础值、主属性类型、growth 引用，全在 Job YAML 里
- **growth 配置独立**：每级增长量单独文件，可被多个 Job 复用（如 warrior 和 swordsman 可能用同一套 growth）
- **全局规则独立**：自由点/技能点发放规则、属性换算公式
- **玩家数据存 Metadata**：只存分配量和余额，最终值实时计算

## 六、枚举定义（不定义 ClassType）

只定义属性维度，**不定义职业枚举**：

```kotlin
// 新增
enum class PrimaryAttribute(val displayName: String, val shortName: String) {
    STRENGTH("力量", "STR"),
    INTELLIGENCE("智力", "INT"),
    AGILITY("敏捷", "AGI"),
    ENDURANCE("耐力", "END");
}

enum class SubAttribute(val displayName: String, val shortName: String) {
    CRIT("暴击", "CRIT"),
    STRIKE("强击", "STRIKE"),
    FAITH("信仰", "FAITH"),
    HASTE("急速", "HASTE"),
    TOUGHNESS("坚韧", "TOUGH"),
    SPIRIT("精神", "SPIR");
}
```

职业身份由 `ImmutableJob.id` 和 `main-attribute` 字段决定，不做硬编码枚举。

## 七、PlayerTemplate Metadata 扩展

```
attribute.main-attribute     → "STRENGTH"
attribute.invested.STRENGTH  → 15
attribute.invested.CRIT      → 3
attribute.free_points        → 7
attribute.class_skill_points → 12
attribute.war_gauge          → 0
```

- 只存可变数据（分配量、余额）
- 最终属性值实时计算：`Job基础值 + growth × (level-1) + invested + 装备 + Buff`
- `main-attribute` 缓存在 metadata 避免每次查 Job（PlayerRoute 可能变化）

class ImmutableJob(private val config: Configuration) : Job {
    // 现有字段不变 ...

    /** 主属性类型 */
    val mainAttribute: PrimaryAttribute?
        get() = config.getString("__option__.main-attribute")
            ?.let { PrimaryAttribute.valueOf(it) }

    /** Lv1 基础主属性 */
    val basePrimary: Map<PrimaryAttribute, Int>
        get() { /* 解析 __option__.attributes.primary */ }

    /** Lv1 基础子属性 */
    val baseSub: Map<SubAttribute, Int>
        get() { /* 解析 __option__.attributes.sub */ }

    /** Lv1 基础资源 */
    val baseResource: Map<String, Int>
        get() { /* 解析 __option__.attributes.resource */ }

    /** growth 配置 id */
    val growthId: String?
        get() = config.getString("__option__.growth")
}

## 九、与现有模块集成

### 9.1 升级发放 — 监听现有事件

```kotlin
object ClassAttributeListener {

    @SubscribeEvent
    fun onLevelChange(e: PlayerLevelChangeEvent) {
        ClassAttributeManager.onLevelUp(e.template, e.to)
        AttributeProvider.refresh(e.player)  // 触发属性同步
    }

    @SubscribeEvent
    fun onRouteSet(e: PlayerSetRouteEvent.Post) {
        val job = e.route.getJob() as? ImmutableJob ?: return
        // 首次选 Job 且有 growth → 初始化属性
        if (job.growthId != null && 
            e.template["attribute.main-attribute"] == null) {
            ClassAttributeManager.init(e.template, job)
        }
        AttributeProvider.refresh(e.player)
    }
}
```

### 9.2 属性同步 — 扩展现有 AttributeProvider

```kotlin
// 在现有 getAttributeList() 后追加
private fun getAttributeListWithClassGrowth(player: Player): List<String> {
    val template = player.plannersTemplate
    val list = mutableListOf<String>()

    PrimaryAttribute.entries.forEach { attr ->
        val v = ClassAttributeManager.getPrimary(template, attr)
        list.add("${attr.shortName}: +$v")
    }
    SubAttribute.entries.forEach { attr ->
        val v = ClassAttributeManager.getSub(template, attr)
        list.add("${attr.shortName}: +$v")
    }
    list.add("HP_MAX: +${ClassAttributeManager.getMaxHP(template)}")
    list.add("MP_MAX: +${ClassAttributeManager.getMaxMP(template)}")
    list.add("SP_MAX: +${ClassAttributeManager.getMaxSP(template)}")
    return list
}
```

## 十二、边界约束

- 不存储最终属性值，运行时计算
- 武器技能点由武器熟练度系统独立管理，不在此模块
- 装备/Buff 属性叠加由各自的 AttributeProvider 段处理，本模块只负责 Job 基础 + growth + invested
- 洗点只返还自由属性点，不触及技能树
- ULP 固定 500
- 无"闪避"子属性
